# Изменения — Спринт 2026-08

Партия правок по замечаниям заказчика. Шесть независимых деплоев (Д1–Д6), все в `master`.
Технические детали — в сообщениях коммитов; ниже — функциональное описание.

---

## Д1 — Статусы и нумерация заявок
- Рабочих статуса теперь три: **Новая → Принята → Доставлена**, плюс отдельный
  статус **Отменена**.
- Раздельная нумерация заявок для физлиц, юрлиц и внутренних (ТТН-Л).
- Добавлен тип внутренней заявки **ТТН-Л** (создаёт только менеджер).

## Д2 — Каталог топлива
- Виды топлива вынесены в справочник-таблицу (менеджер ведёт сам), вместо
  жёсткого списка в коде.
- Клиент при создании заявки видит **только топливо, которое есть в наличии**.
- Водителю показывается остаток на складе.
- Минимальный объём заказа — **300 литров**.

## Д3 — Зоны доставки
- Менеджер рисует зоны доставки на карте (Leaflet + geoman).
- Адрес заявки геокодируется через DaData; автоматически определяется зона
  (проверка «точка внутри полигона»).
- У каждой зоны — свой коэффициент, влияющий на стоимость доставки.

## Д4 — Документы (счета)
- Клиенту выставляются **два счёта**: предварительный (при создании) и
  итоговый (по факту доставки).
- Основание счёта — **договор поставки** (без отдельной строки доставки в счёте).
- Заявки от **3000 л** не выставляются автоматически — менеджер получает
  уведомление и выставляет счёт вручную.

## Д5 — Тарифы и оплаты
- Два базовых тарифа — для **физлиц** и для **юрлиц**; клиент по типу получает
  свой тариф по умолчанию, для конкретного клиента можно переопределить.
- **Долговая заявка** (доставка без оплаты): ставит только менеджер. Водителю
  долг не виден — для него заявка выглядит как «можно доставлять».
- Водитель фиксирует оплату **только у физлиц** (с всплывающим окном суммы
  после отметки «Доставлена»); у юрлиц денег вообще не видит.
- Клиенту при неоплаченной заявке — крупное оповещение «Заявку необходимо
  оплатить. Свяжитесь с менеджером».

## Д6 — Вход и SMS
- Упрощённая регистрация физлица: **телефон, имя, пароль**; email
  необязателен, дозаполняется в личном кабинете.
- Вход по **единому идентификатору** — email **или** телефон + пароль.
- Вход по **SMS-коду** и **сброс пароля по SMS** (коды через SMSC.ru,
  живут ограниченное время, с защитой от подбора и перебора номеров).
- Админ может редактировать данные любого клиента (контакты, паспорт,
  реквизиты). Поля из DaData обновляются только автосинхронизацией.

## Что нужно для деплоя
1. Прописать в `.env` ключи SMSC.ru (`SMSC_LOGIN`, `SMSC_PASSWORD`,
   `SMSC_SENDER`) и включить `SMS_ENABLED=true` — иначе SMS молча не отправляются.
2. Прогнать миграции: `alembic upgrade head` в `order_service` и `auth_service`.
3. Перезапуск с пересозданием контейнеров при изменении `.env`
   (`--force-recreate`), иначе новые переменные не подхватятся.
