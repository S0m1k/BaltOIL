# SPEC — Спринт 2026-07 (договор, доверенность, цветной реестр, единый источник реквизитов, редизайн PDF)

**Контекст.** Заказчик 2026-05-26 прислал три материала:
1. Образец договора поставки нефтепродуктов (`обр договора.docx`).
2. Образец счёта (`Обр счета.xls`) с цветовой разметкой зон: **жёлтое** = реквизиты продавца (редактируются в одном месте, автоподтягиваются во ВСЕ документы), **красное** = изменяющиеся данные (номер, дата, данные клиента), **зелёное** = вычисляемое (суммы, прописью).
3. Скрин реестра заявок: зелёные строки = оплачено, белые = неоплачено, жёлтые = частичная оплата, красные = отменено (причина в комментарии).
4. PDF `Реквизиты ООО ЛИДЕР ДИЗ.pdf` — настоящие реквизиты продавца для seeding.

**Решения, зафиксированные при планировании (Opus, 2026-05-26).**
- **Скоуп = вариант (c)**: договор + доверенность + цветной реестр + редизайн шаблонов счёта/ТТН/УПД под образец.
- **Договор без подписи** (печать/факсимиле не закладываем — текстовое место под подпись).
- **Триггер генерации договора — оба варианта**: ручной из карточки клиента + автоматический при первой заявке клиента (если активного договора ещё нет).
- **Реквизиты продавца** — UI без печати/факсимиле, только поля. Источник один: `LegalEntity` (модель уже есть с версионированием).
- **Реестр** — цветим существующую таблицу заявок, не делаем отдельную страницу.
- **История документов в карточке клиента** — отдельная вкладка/секция.
- Реквизиты ООО «Лидер Диз» из PDF идут в seed-миграцию (если заказчик подтвердит, что это и есть продавец).

**Принципы.**
- BaltOIL — master only. Без feature-веток.
- 4 деплоя, каждый катится независимо. Все миграции идемпотентные, по 1 на деплой.
- `.env` на проде — не трогаем (только `.env.example`).
- Untracked на сервере не трогаем.

  ```python
  class Contract(Base):
      __tablename__ = "contracts"
      id: UUID PK
      client_id: UUID, FK (auth user), index
      contract_number: str(20), unique  # "NNN/MM" формат, напр. "034/02"
  ```

- Миграция `00XX_contracts.py` — создание таблиц `contracts`, `contract_counters`. Idempotent (`CREATE TABLE IF NOT EXISTS`).
